# JVM Memory Management

## Introduction

Java provides **Automatic Memory Management** through the **Java Virtual Machine (JVM)**.

When a Java program is executed, the **Java Launcher (`java` command)** creates and starts a **JVM instance**.

The JVM requests memory from the **Operating System**.

The Operating System allocates a **Virtual Address Space** for the JVM process.

The JVM uses this Virtual Address Space to organize its memory areas.

---

## Memory Allocation Process

```text
Java Launcher
      │
      ▼
JVM Instance
      │
      ▼
Operating System
      │
      ▼
Virtual Address Space
      │
      ├── Heap
      ├── Java Stack
      ├── Metaspace
      ├── PC Register
      └── Native Method Stack
```

---

# Java Stack

Each thread has its own **Java Stack**.

### Stores

* Method Calls (Stack Frames)
* Local Variables
* Primitive Data Types
* Object References

### Characteristics

* Follows **LIFO (Last In, First Out)**.
* Stack Frames are automatically removed after method execution.
* If the Stack exceeds its limit, a **StackOverflowError** occurs.

---

# Heap Memory

Heap stores:

* Objects
* Arrays

### Characteristics

* Shared by all threads.
* Objects created using the **`new`** keyword are stored in the Heap.

---

# Metaspace

Metaspace stores:

* Class Metadata
* Method Information
* Field Information
* Runtime Constant Pool

---

# PC Register

The **PC Register** stores the address of the current instruction being executed by each thread.

Each thread has its own PC Register.

---

# Native Method Stack

The Native Method Stack stores information required to execute native methods written in languages such as C and C++.

---

# Key Points

* Java Launcher creates and starts the JVM instance.
* JVM requests memory from the Operating System.
* The Operating System provides a Virtual Address Space.
* JVM organizes memory into Heap, Java Stack, Metaspace, PC Register, and Native Method Stack.
* Each thread has its own Stack.
* All threads share a single Heap.
* Objects are stored in the Heap.
* Object references are stored in the Stack.
* Stack follows the LIFO principle.
