# Garbage Collector (GC)

## Introduction

The **Garbage Collector (GC)** is a component of the JVM that automatically manages Heap memory.

Its primary responsibility is to reclaim memory occupied by objects that are no longer reachable.

---

## Why Garbage Collector?

In languages such as **C and C++**, memory allocated in the Heap must be managed manually.

In Java, the Garbage Collector automatically reclaims unused Heap memory, reducing memory leaks and simplifying memory management.

---

## How Garbage Collector Works

Objects created using the **`new`** keyword are stored in the Heap.

The object reference is stored in the Stack (if it is a local variable).

When a method finishes execution, its Stack Frame is removed.

If an object no longer has any **live references**, it becomes **eligible for Garbage Collection**.

The Garbage Collector later reclaims the memory occupied by that object.

---

## Example

```java
Student s = new Student();
```

* `Student` object → Heap
* `s` reference → Stack

When the method ends, the reference is removed from the Stack.

If no other live reference points to the object, it becomes eligible for Garbage Collection.

---

## Manual vs Automatic Memory Management

| C / C++                     | Java                              |
| --------------------------- | --------------------------------- |
| Manual Memory Management    | Automatic Memory Management       |
| Programmer releases memory  | Garbage Collector reclaims memory |
| Higher risk of memory leaks | Reduced memory leaks              |

---

## Key Points

* Garbage Collector manages Heap memory.
* It removes objects that are no longer reachable.
* An object with no live references becomes eligible for Garbage Collection.
* Stack memory is automatically released after method execution.
* Java provides Automatic Memory Management through the Garbage Collector.
