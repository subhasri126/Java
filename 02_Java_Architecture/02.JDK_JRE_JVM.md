# JDK, JRE and JVM

## Introduction

JDK, JRE, and JVM are the three main components of Java. They work together to develop, compile, and execute Java programs.

---

## Relationship

```text
JDK
 └── JRE
      └── JVM
```

---

# JDK (Java Development Kit)

JDK is used to **develop Java applications**.

It contains all the tools required to write, compile, debug, and execute Java programs.

### Components of JDK

* javac (Java Compiler)
* java (Java Launcher)
* jdb (Java Debugger)
* jps (Java Process Status Tool)
* JRE

---

## What Happens During Compilation?

When the `javac` compiler compiles a Java program, it performs the following steps:

### 1. Abstract Syntax Tree (AST)

The compiler analyzes the source code and creates an **Abstract Syntax Tree (AST)**.

The AST represents the structure of the program and helps the compiler understand the syntax.

---

### 2. Symbol Table

The compiler creates a **Symbol Table**.

The Symbol Table stores information about:

* Variables
* Methods
* Classes

It is also used during syntax and semantic checking.

---

### 3. Error Checking

The compiler checks the program for syntax errors.

If any errors are found, the compilation process stops.

---

### 4. Bytecode Generation

If no errors are found, the compiler generates a **Bytecode (.class)** file.

This Bytecode is platform independent.

---

# JRE (Java Runtime Environment)

JRE provides the **runtime environment** required to execute Java programs.

If a Java program is already compiled into Bytecode (`.class`), only the **JRE** is required to run it.

### JRE Contains

* JVM
* Java Libraries
* Runtime Files

---

# JVM (Java Virtual Machine)

JVM is responsible for executing Java programs.

It reads the Bytecode generated by the compiler and converts it into Machine Code.

### Responsibilities of JVM

* Loads classes.
* Executes Bytecode.
* Converts Bytecode into Machine Code.
* Executes the program.

---

## Key Points

* JDK is used to develop Java applications.
* JDK contains the JRE.
* JRE provides the runtime environment.
* JRE contains the JVM.
* The `javac` compiler creates the **AST** and **Symbol Table** during compilation.
* The compiler generates the **Bytecode (.class)** file.
* JVM executes the Bytecode and converts it into Machine Code.
* `javac` → Compiler
* `java` → Runs Java programs
* `jdb` → Debugger
* `jps` → Shows running Java processes
