# JIT (Just-In-Time) Compiler

## Introduction

The **JIT (Just-In-Time) Compiler** is a component of the **JVM**.

Its main purpose is to improve the execution speed of Java programs.

Instead of interpreting the same code repeatedly, the JIT Compiler converts frequently executed code into native machine code for faster execution.

---

## Why is JIT Required?

Normally, the JVM interprets Bytecode line by line.

If a method is executed many times, interpreting it repeatedly reduces performance.

The JIT Compiler solves this problem by compiling frequently executed methods into native machine code.

---

## Hot Methods

A **Hot Method** is a method that is executed many times during program execution.

The JIT Compiler identifies these methods and compiles them only once.

Later executions use the compiled machine code directly, improving performance.

---

## Basic Optimizations

### 1. Method Inlining

Method Inlining reduces the overhead of repeatedly calling small methods.

Instead of calling the method again and again, the JIT Compiler may insert the method's code directly where it is used.

---

### 2. Loop Unrolling

Loop Unrolling improves loop performance by reducing the number of loop iterations and loop-control operations.

This helps certain loops execute faster.


---

## Key Points

* JIT stands for **Just-In-Time Compiler**.
* JIT is a component of the JVM.
* JIT improves the performance of Java programs.
* JIT identifies **Hot Methods**.
* Frequently executed methods are compiled into native machine code.
* Basic optimizations include **Method Inlining** and **Loop Unrolling**.
