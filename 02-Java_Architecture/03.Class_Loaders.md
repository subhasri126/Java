# Class Loaders

## Introduction

Before executing a Java program, the **JVM** must load all the required classes into memory.

This is done by the **Class Loader Subsystem**.

Java uses three types of Class Loaders.

---

## Class Loader Hierarchy

```text
Bootstrap Class Loader
          │
          ▼
Platform Class Loader
          │
          ▼
Application Class Loader
```

This order is called the **Delegation Hierarchy**.

The JVM first checks the **Bootstrap Class Loader**. If the required class is not found, it checks the **Platform Class Loader**. Finally, it checks the **Application Class Loader**.

---

## 1. Bootstrap Class Loader

The Bootstrap Class Loader loads Java's core built-in classes.

### Examples

* `java.lang`
* `java.util`
* `java.io`

---

## 2. Platform Class Loader

The Platform Class Loader loads Java platform modules and libraries.

### Examples

* `java.sql`
* `java.xml`
* `java.management`

---

## 3. Application Class Loader

The Application Class Loader loads the classes created by the programmer.

### Example

```java
public class Student {
}
```

The `Student` class is loaded by the Application Class Loader.

---

## Delegation Hierarchy

When a class needs to be loaded:

1. Bootstrap Class Loader is checked first.
2. If not found, Platform Class Loader is checked.
3. If still not found, Application Class Loader loads the class.

This process is called the **Delegation Hierarchy**.

---

## Key Points

* JVM uses the Class Loader Subsystem to load classes.
* Java has three Class Loaders.
* Bootstrap Class Loader loads core Java classes.
* Platform Class Loader loads Java platform libraries.
* Application Class Loader loads user-defined classes.
* Class loading follows the Delegation Hierarchy.
