# JVM Working

## Introduction

The **Java Virtual Machine (JVM)** is responsible for executing Java Bytecode. It loads classes, verifies the Bytecode, manages memory, executes the program, and communicates with the operating system.

---

## JVM Working Flow

```text
Bytecode (.class)
        │
        ▼
Class Loader Subsystem
        │
        ▼
Bytecode Verifier
        │
        ▼
Runtime Data Area
        │
        ▼
Execution Engine
     ├── Interpreter
     └── JIT (Just-In-Time) Compiler
        │
        ▼
Java Native Interface (JNI)
        │
        ▼
Native Libraries
        │
        ▼
Operating System
```

---

## 1. Class Loader Subsystem

The **Class Loader Subsystem** loads all the required classes into memory before the program starts execution.

It follows the **Delegation Hierarchy**.

### Bootstrap Class Loader

* Loads Java's core built-in classes.
* Examples:

  * `java.lang`
  * `java.util`
  * `java.io`

### Platform Class Loader

* Loads Java platform libraries and modules.
* Examples:

  * `java.sql`
  * `java.xml`
  * `java.management`

### Application Class Loader

* Loads user-defined classes created by the programmer.

### Delegation Hierarchy

When a class needs to be loaded:

1. Bootstrap Class Loader checks first.
2. If the class is not found, Platform Class Loader checks next.
3. If still not found, Application Class Loader loads the class.

---

## 2. Bytecode Verifier

The **Bytecode Verifier** checks whether the Bytecode is valid and safe to execute.

It verifies:

* Bytecode format
* Illegal instructions
* Security issues

If the Bytecode is valid, the execution continues.

---

## 3. Runtime Data Area

The **Runtime Data Area** allocates memory required for program execution.

It contains:

* Heap Memory
* Stack Memory
* Method Area
* PC Register
* Native Method Stack

---

## 4. Execution Engine

The **Execution Engine** executes the Bytecode.

### Interpreter

* Executes Bytecode line by line.

### JIT (Just-In-Time) Compiler

* Identifies frequently executed (**Hot**) methods.
* Converts them into native machine code.
* Improves the execution speed of Java programs.

---

## 5. Java Native Interface (JNI)

The **Java Native Interface (JNI)** acts as a bridge between Java and native languages such as C and C++.

It allows Java programs to access native code and operating system features.

---

## 6. Native Libraries

Native Libraries contain platform-specific code used by the operating system.

JNI communicates with these libraries whenever native functionality is required.

---

## Key Points

* JVM executes Java Bytecode.
* Class Loader Subsystem loads all required classes using the Delegation Hierarchy.
* Bytecode Verifier checks the validity and security of the Bytecode.
* Runtime Data Area allocates memory for program execution.
* Execution Engine executes the program using the Interpreter and JIT Compiler.
* JNI connects Java with native code.
* Native Libraries provide operating system-specific functionality.
