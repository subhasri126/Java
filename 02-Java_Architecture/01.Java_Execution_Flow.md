# Java Execution Flow

## Introduction

A Java program goes through several stages before it is executed. Java uses both a **Compiler** and a **Java Virtual Machine (JVM)** to execute a program. This is why Java is known as a **Hybrid Programming Language**.

---

## Java Execution Flow

```text
Source Code (.java)
        │
        ▼
Compilation (javac Compiler)
        │
        ▼
Bytecode (.class)
        │
        ▼
JRE (Runtime Environment)
        │
        ▼
JVM
        │
        ▼
Machine Code
        │
        ▼
Program Execution
```

---

## Step 1: Write the Source Code

The Java program is written in a **.java** file.

Example:

```text
HelloWorld.java
```

---

## Step 2: Compilation

The **javac Compiler** (available in the JDK) compiles the source code and generates a **Bytecode (.class)** file.

---

## Step 3: Bytecode Generation

The generated **Bytecode** is platform independent.

This Bytecode can run on any operating system that has a compatible **JVM**.

---

## Step 4: Runtime Environment

The **JRE (Java Runtime Environment)** provides the environment required to run the Java program.

---

## Step 5: JVM Execution

The **JVM (Java Virtual Machine)** reads the Bytecode, converts it into Machine Code, and executes the program.

---

## Why is Java Called a Hybrid Language?

Java uses:

* **Compiler** – Converts Source Code into Bytecode.
* **JVM (Interpreter)** – Executes the Bytecode.

Therefore, Java is called a **Hybrid Programming Language**.

---

## Key Points

* Java source code is stored in a **.java** file.
* The **javac Compiler** converts source code into **Bytecode (.class)**.
* Bytecode is platform independent.
* JRE provides the runtime environment.
* JVM executes the Bytecode.
* Java uses both a Compiler and a JVM (Interpreter), so it is called a **Hybrid Programming Language**.
